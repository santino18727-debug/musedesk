# Audit complet MuseDesk — 2026-06-27

Audit multi-agents (via agent-finder) en **lecture seule**, 4 dimensions :
sécurité (`security-reviewer`), robustesse infra (`infra-homelab-doctor`),
qualité code/bugs/perf (`Code Reviewer`), accessibilité WCAG (`Accessibility Auditor`).

## Verdict global

**Base saine, déploiement et garanties à durcir.** Le code applicatif est au-dessus de
la moyenne d'un projet perso (échappement XSS centralisé, CSP sans `unsafe-inline` sur
`script-src`, token CSPRNG 128 bits, OAuth éphémère scope minimal, relais zéro-persistance,
serveur read-only côté followers). **Aucun secret commité.** Pas de faille critique de sécurité.

MAIS trois angles faibles réels :
1. **L'enveloppe de déploiement du relais** (pas de limite RAM → risque OOM, Funnel trop large, zéro monitoring).
2. **La garantie « lecture seule » du follower est incomplète** (ses inputs mutent encore son état → désync).
3. **Quelques bugs réseau** (double connexion WS, race de purge de session) + **WCAG AA non conforme** (5 bloquants).

## 🔴 CRITIQUES (à corriger en priorité)

### Infra (ta main — prod T450)
- **F1 — Conteneur relais SANS limite RAM → rejoue l'OOM du 2026-05-31.** `docker inspect` = `Memory=0`. Le `mem_limit:128m` n'existe que dans le compose (non utilisé). Sur 7.5 Gi déjà à 1.2 Gi libre, une fuite/afflux peut déclencher l'OOM-killer qui tuera qdrant/ollama.
  → **Fix** : relancer le conteneur avec `--memory=128m --memory-reservation=32m`.
- **F2 — Funnel expose `/` entier (pas `/relay`).** `tailscale funnel status` = `/ → 127.0.0.1:18800`. Le `/relay/health` public divulgue `sessions.size`. Viole SEC-01 (ADR-045). Le README ment (parle de Caddy `/relay/*` inexistant).
  → **Fix** : scoper le funnel au sous-chemin `/relay`, ou au minimum retirer `sessions.size` du health public.

### Code (je peux corriger)
- **B6 — Le follower n'est PAS réellement en lecture seule.** `app.js` : tap-zones, swipe, flèches clavier, scroll, transpose, 2-col, font restent actifs en mode follower → un tap désynchronise. La « lecture seule » promise n'est pas garantie.
  → **Fix** : `if (isFollower()) return;` en tête de chaque handler d'entrée.
- **B2 — Double connexion WS.** `live.js:118` : `openWs()` ne détache pas les handlers de l'ancienne socket → l'ancien `onclose` replanifie une 2ᵉ reconnexion → 2 sockets + 2 timers, compteur `peers` faussé.
  → **Fix** : `ws.onopen=ws.onmessage=ws.onerror=ws.onclose=null` + `cancelReconnect()` avant réassignation.
- **B4 — Race de purge de session (relais).** `server.js:121` : le `graceTimer` supprime la session même si des followers sont encore connectés (`!cur.leader` seul) → followers actifs orphelins, snapshot perdu.
  → **Fix** : `if (cur && !cur.leader && cur.followers.size === 0) sessions.delete(...)`.
- **B5 — Fuite IndexedDB en follower.** `app.js:920` : `concertNext/Prev` appellent `db.getSetlist` direct (pas l'adaptateur `getSetlistForRender`) → navigation concert cassée/désync côté follower.
  → **Fix** : router via `getSetlistForRender`.
- **B3 — `_reconnectTimer` non annulé** quand `createSession`/`joinSession` rappelés → `openWs()` parasite avec nouveaux role/token.
  → **Fix** : `cancelReconnect()` + `stopPing()` en tête.

### Accessibilité (je peux corriger — 5 bloquants → WCAG AA non conforme)
- **A1 — Sliders inutilisables au clavier** : le handler global flèches `preventDefault()` vole les `Arrow*` même quand le focus est sur un `<input range>`. → `if (e.target.matches('input,textarea,select')) return;` (1 ligne, ROI max).
- **A2 — Pas de retour de focus après fermeture des dialogs** → utilisateur clavier éjecté en haut de page. Focus initial absent dans settings/pupitre.
- **A3 — Réorganisation de setlist 100% drag souris** → impossible au clavier/AT. → ajouter boutons ↑/↓ par ligne.
- **A4 — Barre alphabet en `<a href="#">`** : 26 « liens » dont la moitié inertes, cibles ~10px. → `<button>` + `disabled` pour lettres absentes.
- **A5 — Changements d'état non annoncés** (scroll/2col/transpose/capo/`pupitre-peers`) : pas d'`aria-pressed`, pas de live region.

## 🟠 SÉRIEUX

### Sécurité
- **SEC1 — Relais sans `Origin`-check ni rate-limit** : n'importe quel site tiers peut scripter des connexions WS et saturer `MAX_SESS=500`. → `verifyClient` (allowlist Origin Pages+localhost) + token-bucket par IP.
- **SEC2 — Vol de rôle leader pendant la grâce 120 s** : qui a vu le QR peut récupérer le contrôle live à la déco du leader. → nonce de continuité leader au `welcome`.
- **SEC3 — pdf.js 4.7.76** : patchée contre les CVE connues (CVE-2024-4367) mais ~15 mois de retard ; cible XSS/RCE récurrente. → vérifier advisories postérieurs + bumper + `npm audit` sur le relais.
- **SEC4 — Snapshot setlist (paroles/accords, possible droits d'auteur) transite par le T450** (chiffré, éphémère, zéro disque). Point de gouvernance RGPD si diffusion hors cercle privé.

### Infra
- **F3 — Persistance Funnel au reboot non prouvée** (posé à la main, aucune unit systemd, T450 uptime 36 j → jamais retesté). → tester un reboot à froid hors-concert ; unit systemd `oneshot` si non persistant.
- **F4 — Dérive `docker run` vs compose** : la source de vérité (README+compose) ne décrit pas la prod réelle. → réconcilier ou documenter la commande exacte.
- **F5 — Zéro monitoring du relais** : Uptime Kuma (12 sondes) n'en a aucune sur le relais. Panne en concert = découverte quand les pupitres décrochent. → 2 sondes Kuma : interne `:18800/relay/health` + publique `:8443/relay/health`.

### Code / perf
- **S2 — Sync `last-write-wins` sans tombstones** : un morceau supprimé sur A « ressuscite » s'il existe encore sur B. Perte/résurrection silencieuse. → `deletedAt` soft-delete.
- **S3 — Push de TOUTE la bibliothèque à chaque sync** (même sans changement, toutes les 2,5 s après chaque action). → ne pousser que si `dirty`/`updatedAt > lastPushAt`.
- **S4 — Tests relais : edge cases critiques non couverts** (reconnexion-leader-grâce = le cœur de la feature, `too-many-peers`, purge B4, close 1009). → ajouter au moins le test reconnexion-grâce.
- **S5 — `keyOf`/`_keyCache` sans borne** → fuite mémoire lente (clé = contenu entier). → clé `id:updatedAt` ou LRU borné.
- **S6 — SW cache-first sans network pour `index.html`/`./`/`manifest.json`** (non versionnés) → nouvelle index.html jamais récupérée sans bump CACHE. → network-first sur ces 3.
- **S7 — Footgun `?v=N` dupliqué manuellement** : divergence possible → deux instances de `db.js` (2 connexions IndexedDB). → centraliser la version, ou check CI qui grep les `?v=`.
- **S1 — Fuite de listeners** : `openPickerDialog` ré-attache un `input` à chaque ouverture. → `oninput=` ou attacher une fois.

### Accessibilité
- **A6 — Contrastes sous AA** : `--text-3 #7a6c56` = 3.6:1 (placeholders/badges/métadonnées), texte blanc sur boutons terracotta = 3.21:1 (CTA principaux), poignée drag 2.0-2.4:1. → éclaircir `--text-3` à ~4.5:1 ; texte des `.btn-primary` en `--accent-ink` sombre.
- **A7 — Cibles tactiles <24px** (alphabet, favori, ✕ setlist, mini live bar) — critique pour le cas « musicien debout sur scène ». → 44px de hit-area.
- **A8-A11** — pagination clavier non découvrable, popover accords souris-only, métronome/toggles sans live region/`aria-pressed`, QR `<img>` sans `alt` dans conteneur labellé.

## 🟡 / ⚪ (notables, non bloquants)
- `escapeHTML` n'échappe pas `'` → robustesse défensive (fragile si futur attribut single-quote). [sécu + code J6]
- B1 — reconnexion sans **jitter** → N followers se reconnectent en même temps (pics synchronisés sur petit T450). → jitter aléatoire.
- F6 — back-pressure follower lent non gérée (`f.send` sans check `bufferedAmount`) ; borné par MAX_PEERS+heartbeat, vrai filet = F1 (RAM).
- F7 — `/relay/health` fuite `sessions.size` (réglé par F2).
- J1 — throttle scroll sans trailing edge (follower figé ~150 ms avant la position réelle).
- J2 — `detectKey` rappelé non-mémoïsé dans le lecteur (utiliser `keyOf`).
- J3 — `renderSong` reconstruit tout l'innerHTML à chaque micro-changement (flash sur tablette).
- A12-A18 — landmarks/`<main>`/skip-link, `aria-labelledby` dialogs, auto-scroll JS ignore `reduced-motion`, labels recherche/tri, `lang` sur paroles.
